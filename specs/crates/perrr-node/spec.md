## Purpose
- Single napi-rs binding layer between Rust core and Node.js V8.
- All JS↔Rust traffic flows through this crate.
- No business logic; thin wrapper that delegates to downstream crates (`perrr-dom`, `perrr-style`, …) in future milestones.

## Contract
### MUST
- Crate type: `["cdylib", "rlib"]`.
- Export `hello() -> "ok"` via `#[napi]` (M1 smoke test).
- Export `PerrrDom` class wrapping `perrr_dom::Tree`; one instance per Vitest test context.
- All panics caught at N-API boundary; translated to `napi::Error` → `JsError`.
- Handle-based API: cross-boundary node references are `u32` `NodeId`; never pointers, refs, or `Box<T>`.
- `DomError` variants surface as `napi::Error` with message preserved.
- Build via `@napi-rs/cli build`; artifact = single `.node` per `{os, arch, libc}` triple.
- TS types in `packages/perrr/index.d.ts` generated by napi-rs.

### MUST NOT
- Depend on any `perrr-*` crate other than `perrr-dom` at M2. Style / layout / paint / scheduler / perf crates remain plumbing-only until their milestones land.
- Introduce Rust deps beyond `napi`, `napi-derive`, `napi-build`, `perrr-dom`, and `perrr-dom`'s transitive deps.
- Hold mutable global state (state lives on `PerrrDom` instances; passed by `NodeId`).
- Expose raw Rust types to JS; use `#[napi(object)]` or string-keyed dicts at the boundary.

### Invariants
- A Rust panic never unwinds into V8 (`napi-derive` wraps entry points in `catch_unwind`).
- `hello()` is side-effect-free.
- N-API ABI version pinned per release; compat asserted in CI.
- Every `PerrrDom` method is a thin delegator to `perrr_dom::Tree`; no logic in this crate beyond argument translation.

## Non-goals
- No style / layout / paint / scheduler / perf / thrash logic (downstream crates, M3+).
- No async ops at M2.
- No `Buffer` / `TypedArray` interop at M2.

## Design
- `src/lib.rs`: `hello()` + `#[napi] struct PerrrDom { tree: Tree }` with methods wrapping `Tree` ops.
- `build.rs`: `napi_build::setup()`.
- Per-instance state (no globals). Each test context holds one `PerrrDom`; JS-side lifetime managed by GC + explicit teardown.

## Exported surface
- Free fn: `hello() -> string`.
- `PerrrDom` class methods (all camelCased by napi-rs; per-instance state, no globals):
  - ids: `documentId()`, `ownerDocument(n)`, `rootNode(n)`, `activeElement()`
  - create: `createElement`, `createElementNs`, `createTextNode`, `createComment`, `createDocumentFragment`
  - meta: `nodeType`, `localName`, `tagName`, `namespaceUri`, `nodeName`
  - attrs: `getAttribute`, `hasAttribute`, `setAttribute`, `removeAttribute`, `attributeNames`, `idAttr`
  - walk: `parentNode`, `parentElement`, `children`, `firstChild`, `lastChild`, `nextSibling`, `previousSibling`, `contains`
  - mutate: `appendChild`, `insertBefore`, `removeChild`, `freeNode`
  - text: `textContent`, `setTextContent`, `nodeData`, `setNodeData`
  - selectors: `matches`, `querySelector`, `querySelectorAll`, `closest` (added 4e.i; parse errors surface as `napi::Error` with `Status::InvalidArg`)
  - focus: `focus`, `blur`
  - metric: `incrListener`, `decrListener`, `listenerCount`, `totalListenerCount`
- `#[napi(object)] AttrRecord { name, value }` DTO (reserved for future `getAttributeNode`-style ops).
